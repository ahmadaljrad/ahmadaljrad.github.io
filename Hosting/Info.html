الدليل الكامل لاستضافة موقع ahmadaljarad.com

عبر GitHub Pages، ربط نطاق مخصص، وتفعيل حماية SSL

1. تصميم وبناء الموقع

  • هيكلية الملفات الأساسية:
    • index.html ← الصفحة الرئيسية
    • style.css ← التنسيقات والجمالية
    • app.js ← التفاعلية و **الربط مع EmailJS**
    • data.json ← بيانات السيرة الذاتية
  • التقنيات المستخدمة: HTML5، CSS3، JavaScript الحديثة.
  • أبرز المميزات: عناصر تفاعلية، و **نموذج اتصال مفعل** يرسل الرسائل مباشرة عبر خدمة EmailJS.

2. الرفع إلى مستودع GitHub

  1. أنشئ مستودعاً جديداً (Repository) بالاسم الدقيق التالي: 
    ahmadaljarad.github.io
  2. ارفع ملفاتك المحلية باستخدام واجهة السطر البرمجي (CLI): 
    # إضافة كافة الملفات للتجهيز
git add .

# تسجيل التغييرات برسالة توضيحية
git commit -m "Initial commit - Resume version 1.0"

# دفع الملفات إلى الفرع الرئيسي على جيتهاب
git push origin main
  3. فعل **GitHub Pages** من إعدادات المستودع (Settings > Pages):
    • الفرع (Branch): `main`
    • المجلد: `/` (الجذر - root)
  4. سيكون موقعك متاحاً مؤقتاً على: ahmadaljarad.github.io

3. ربط النطاق المخصص aljrad.com

3.1 إعداد سجلات DNS (IPv4)

قم بزيارة لوحة تحكم النطاق لدى مسجل النطاقات الخاص بك وأضف السجلات التالية:

النوع (Type) الاسم/المضيف (Name/Host) القيمة (Value) - GitHub IPs
A @ 185.199.108.153
A @ 185.199.109.153
A @ 185.199.110.153
A @ 185.199.111.153

إعداد سجل CNAME (للـ www)

النوع الاسم/المضيف القيمة (Value)
CNAME www ahmadaljarad.github.io
ملاحظة الانتشار: قد تستغرق تغييرات DNS وقتاً يتراوح من ساعة إلى عدة ساعات لتنتشر حول العالم وتصبح فعالة.

4. تفعيل حماية SSL وتشفير HTTPS عبر Cloudflare

  1. أنشئ حساباً على Cloudflare وأضف نطاقك aljrad.com.
  2. سيقوم Cloudflare باستيراد سجلات DNS تلقائياً. تأكد من مطابقتها لما في الخطوة السابقة.
  3. حدث **خوادم الأسماء (Nameservers)** لدى مسجل نطاقك إلى خوادم Cloudflare الجديدة التي زودوك بها.
  4. في قسم SSL/TLS في Cloudflare، فعل الوضع: **Full** أو **Flexible**.
  5. الآن، موقعك آمن ويمكن الوصول إليه عبر: https://aljrad.com

5. ملاحظات هامة للتهيئة النهائية

  • لتحويل الزوار تلقائياً من `www.aljrad.com` إلى النطاق الجذري، استخدم **Page Rules** في Cloudflare.
  • تأكد أن جميع الروابط الداخلية داخل موقعك تستخدم الترويسة `https://` بعد تفعيل SSL.
  • فعل خاصية **"Automatic HTTPS Rewrites"** في Cloudflare لتجنب مشاكل "المحتوى المختلط" (Mixed Content).
  • اختبر الموقع على أجهزة متصفحات مختلفة (جوال، حاسوب) بعد اعتماد التغييرات.
